Текущая версия: 1.8.1

Версия 1.8.1:
- Ускорен запуск программы.


Версия 1.8:
- Внедрена система автообновления.
- Приписка "для разработчиков" изменена на "для опытных 
пользователей".
- Внесены небольшие изменения в справке.
- (Для опытных пользователей) Параллельно с запуском 
программы, происходит проверка на наличие новой версии 
программы. Новые версии всегда выкладываются на 
личном аккаунте разработчика в GitHub. Если была 
обнаружена новая версия, то программа скачает ее, и 
при следующем запуске выполнит автоматическую 
установку.


Версия 1.7.1:
- Убран подпункт "Открыть логи" в разделе "Помощь".
- Убрано автозаполнение сетевых учетных данных в меню 
"Изменить сетевые учетные данные".


Версия 1.7:
- Добавлен экран загрузки.
- Теперь можно запустить только 1 экземляр 
программы.
- Оптимизирована работа программы.
- Внесены небольшие изменения в справке.
- Добавлен подпункт "Открыть логи" в разделе 
"Помощь".
- Исправлен баг, при котором нельзя было открыть 
файлы любого формата, кроме exe.
- Добавлено уведомление об успешном импорте файла 
конфигурации.
- (Для опытных пользователей) Проверка изменения 
файла конфигурации на сервере теперь 
осуществляется параллельно с ходом работы программы. 
При импорте, файл конфигурации проходит проверку на 
валидность, которая также осуществляется параллельно. 
Если импортируемый файл валиден, то он копируется во 
внутреннюю директорию программы и будет 
использоваться уже при следующем запуске программы.


Версия 1.6:
- Внедрена система логгирования.


Версия 1.5:
- Все файлы, предназначенные для хранения в 
пользовательской папке Документы, теперь хранятся 
внутри специальной папки программы. Пользовательская 
папка Документы более не будет использоваться.
- Теперь программа выгружает файл конфигурации из 
сервера только в том случае, если файл конфигурации был 
изменен на сервере. В остальных случаях файл 
конфигурации выгружается локально.
- (Для опытных пользователей) Служебный файл 
программы теперь имеет параметр, определяющий версии 
программ, которые могут работать с данным 
служебным файлом. Если служебный файл программы 
не поддерживает данную версию программы, то 
программа перепишет имеющийся служебный файл, 
выставив значения параметров по умолчанию.


Версия 1.4.1:
- Небольшие исправления в подпункте "Справка" раздела 
"Помощь".
- Добавлен .bat-файл, позволяющий совершить тихую 
деинсталляцию приложения.


Версия 1.4:
- Скрыта сетевая ошибка 1 при открытии файлов и папок.
- Исправлена проблема, при которой невозможно было 
открыть файлы или папки на локальном устройстве.
- Добавлен параметр "dir_open_timeout" для файлов 
конфигурации. Теперь время подключения к сетевой 
директории определяется не самой программой, а файлом 
конфигурации.
- (Для опытных пользователей) Скорректирована команда 
"explorer <путь_к_папке>", предназначенная для открытия 
папок в проводнике Windows. Теперь команда ищет 
сетевую папку не по имени сетевого устройства, а по его 
IP-адресу.
- (Для опытных пользователей) Команда "net use 
<имя_устройства> /delete" изменена на "net use 
<имя_устройства> /delete /yes".


Версия 1.3:
- Во вкладке "Описание обновлений" теперь указывается 
текущая версия программы.
- (Для опытных пользователей) Скорректирована команда 
"net use <имя_устройства>" и "net use <имя_устройства> 
\delete" для открытия сетевых папок и файлов. Теперь 
эти команды взаимодействует не с именем сетевого 
устройства, а с его IP-адресом, что позволяет 
использовать множественное соединение к данному 
сетевому устройству.


Версия 1.2:
- (Для опытных пользователей) Оптимизировано открытие 
сетевых файлов и папок с использованием команды "net use 
<имя_устройства>", позволяющая открыть сетевые файлы 
и папки сразу с нескольких сетевых устройств.


Версия 1.1:
- Добавлен подпункт "Описание обновлений" в разделе 
меню "Помощь".
- Добавлена возможность запускать исполняемые файлы 
с аргументами.
Например: "\\\\comp1\\folder\\myapp.exe -s -g"
